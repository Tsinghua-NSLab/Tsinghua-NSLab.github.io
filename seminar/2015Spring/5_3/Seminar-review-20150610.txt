Rate-Limiting
If we can find a bound for servicing delay, we can rate- limit hosts so that they never experience queuing delay
1. Network is idle
2. Hosts send ≤ P bytes
3. Wait(nxP/R)secs
4. Goto 1Eliminating Synchronization
network epoch = 2n ⨉ P / R
Where
n - number of hosts
P - bytes sent
R - edge speed
2 - mesochronous compensationKey Idea
Prioritization
Use hardware priorities to run different QJump levels together, but isolated from each other.Conclusions
QJump applies datacenter opportunities to simplify QoS rate calculations.
It provides service levels ranging from guaranteed latency through to line-rate throughput
